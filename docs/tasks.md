## Задание 2 — Кастомный фильтр HasHabits

Сейчас `/list`, `/stats`, `/delete` показывают "нет привычек" текстом внутри handler. Сделай это через **кастомный фильтр** — чтобы handler вызывался только когда привычки есть.

**Шаги:**

1. **Создай файл `filters.py`** — подумай, куда его положить в структуре проекта (рядом с `middlewares.py`? в `handlers/`?)
2. **Изучи код** — найди в handlers, как получают привычки пользователя. Тебе нужен тот же подход
3. **Импортируй всё что нужно** — `BaseFilter`, тип сообщения, модуль хранения
4. **Напиши класс `HasHabits`** — наследуйся от `BaseFilter`, переопредели метод `__call__`
5. **Логика `__call__`** — получи привычки пользователя, верни `True` если они есть, `False` если нет
6. **Подключи фильтр** — вот как это выглядит в декораторе:

```python
from app.filters import HasHabits

@router.message(Command("list"), HasHabits())
async def cmd_list(message: Message) -> None:
    ...  # тут привычки точно есть

# Fallback — сработает когда HasHabits вернул False:
@router.message(Command("list"))
async def cmd_list_empty(message: Message) -> None:
    await message.answer("У тебя пока нет привычек. Добавь через /add")
```

7. **Порядок важен** — handler с фильтром должен быть зарегистрирован **раньше**, чем fallback. aiogram проверяет handlers по порядку

**Как проверить:**
- `/list` без привычек → "нет привычек"
- `/add` → добавь привычку → `/list` → показывает список

## Задание 3 — Дебаггер

Пройдись дебаггером по пути `/list` с твоим фильтром HasHabits. Цель — увидеть своими глазами, как aiogram вызывает фильтр и что в него приходит.

**Подготовка:**
1. Убедись что задание 2 готово и HasHabits работает
2. Добавь через бота хотя бы одну привычку (`/add`)

**Прохождение:**

1. Открой свой `filters.py`, поставь **breakpoint** на первую строку `__call__` в HasHabits
2. Запусти бота через дебаггер: **F5** → "Python Module" → `app`
3. В Telegram напиши боту `/list`
4. VS Code остановится на breakpoint. Теперь:

   **Шаг A** — посмотри что пришло:
   - В панели **Variables** раскрой `message` → `from_user` — найди `id`, `first_name`
   - Там же посмотри `message.text` — должно быть `"/list"`

   **Шаг B** — шагни внутрь:
   - Нажми **F10** (Step Over) до строки с `storage.get_habits()`
   - Нажми **F11** (Step Into) чтобы провалиться внутрь `get_habits`
   - Посмотри как storage читает JSON и возвращает dict
   - Нажми **Shift+F11** (Step Out) чтобы вернуться в фильтр

   **Шаг C** — посмотри результат через Debug Console:
   - Набери: `habits` — увидишь dict с привычками пользователя
   - Набери: `len(habits)` — количество привычек
   - Набери: `bool(habits)` — это то, что вернёт фильтр (`True`)
   - Нажми **F5** (Continue) — бот ответит списком привычек

5. Теперь удали все привычки (`/delete`) и снова напиши `/list`
   - Фильтр вернёт `False` — aiogram вызовет fallback-handler

**Зачем:** дебаггер показывает реальный поток данных. Когда документация непонятна — ставь breakpoint и смотри что приходит.
